What's New

›
v0.4 Feb 2026
  • Fixed: Video Export Truncation Bug in record.js

    Fixed a bug that caused exported videos to be truncated. The issue was in the video export logic in record.js.

  • Feature: Progress Bar on Download Button

    Visual progress indicator that shows export progress percentage while the video is being processed and exported.

  • Added: Feedback button linking to roadmap & feedback portal

    New feedback button that redirects users to a page where they can submit feedback and view the product roadmap.

  • Added: Recording Mode Warning System - warns when non-tab mode is selected

    Warns users that Auto-Zoom and Cursor Overlay only work in Browser Tab mode. Reordered recording options with Browser Tab now first (marked "Recommended"). Added warning box for "Entire Screen" or "Application Window" modes. Auto-Zoom and Show Cursor checkboxes get disabled when non-tab mode is selected. Default recording type changed from "screen" to "tab".

  • Added: Collapsible "What's New" changelog section

    New collapsible section that displays version updates with media to users. Includes toggle arrow and version v0.4 with 4 bullet points.

  • Changed: Zoom Intensity Default (2x → 1.5x) for subtler auto-zoom

    Reduced default zoom intensity from 2x to 1.5x to make auto-zoom more subtle and natural by default.

  • Changed: Animation Duration UI - replaced slider with segmented control

    Changed animation duration control from slider to segmented control buttons (Rapid/Quick/Default/Slow). Replaces old cubic bezier easing with spring physics system.

  • New: Spring Physics System - smoother, more natural animations

    Added createSpring(), ensureZoomSprings(), and ensureCursorSprings() methods. Replaces transition-based animation with stateful spring physics (position + velocity) for smoother, more natural "Screen Studio-like" animation.

  • Bug Fixes: Fixed zoom jitter, added spring snapping on seek

    Fixed zoom jitter by removing duplicate getZoomStateAtTime calls per frame. Added spring snapping on seek to prevent cursor "flyback". Uses currentZoomState directly instead of recalculating.

  • Changed: Real-time Zoom disabled - auto-zoom now works post-recording only

    useRealtimeZoom is now always false - auto-zoom only works post-recording for better performance and reliability.

  • New: Post-recording Auto-zoom - captures clicks during recording and generates zoom segments after

    Added autoZoomClickEvents array to capture clicks during recording. Generates zoom segments from clicks after recording stops.
